完成Cloud Code的安装与配置后，即可开始使用其核心功能。本部分将介绍其基础命令操作、在IDE中的集成方式，并通过一个完整的编程实例演示其解决问题的工作流程。

## 命令行基础操作概述

启动Cloud Code并进入其交互式命令行界面后，您将看到一个命令提示符。在此界面中，所有命令均以正斜杠 `/` 开头，无需再输入 `cloud` 前缀。系统支持命令的智能补全，输入 `/` 后按 `Tab` 键可查看所有可用命令。

### 尝试 Shell 转义符（快捷键）

在许多类似的 CLI AI 工具（如 Aider 或其他的 Agent 终端）中，通用的惯例是使用感叹号 `!` 作为前缀来直接透传 Shell 命令。你可以尝试输入 `!npm install`。如果该版本的 Cloud Code 遵循此惯例，加了感叹号的内容会绕过 LLM 的“思考”环节，直接丢给底层的 Bash/Zsh 执行。

## 常用命令详解

### 清理上下文
命令 `/clear` 用于清空当前对话的上下文历史，相当于开启一个全新的会话。当对话偏离主题或AI响应质量下降时，此命令能够帮助您重新开始对话，确保后续交互的准确性。

### 压缩对话
命令 `/compress` 会对当前对话历史进行总结性压缩，并基于压缩后的上下文开启一个新会话。此功能在需要保留先前对话核心信息的同时，避免冗长的历史记录对新问题分析造成干扰，类似于创建一个对话的副本分支。

### 查看花费
命令 `/cost` 用于统计当前会话累计消耗的费用、API延迟时间以及输入输出的Token数量。对于按使用量计费的账户，此命令有助于监控资源消耗。按月订阅的用户通常无需关注此项数据。统计信息仅针对当前活跃会话，退出或新建会话后会自动清零。

### 登录与登出
命令 `/login` 和 `/logout` 用于切换不同的Cloud服务账号。当需要更换主账号或当前账号额度不足时，可以通过这两个命令方便地完成账号切换操作。

### 切换模型
命令 `/model` 允许用户在Cloud服务提供的不同AI模型之间切换。通常包括响应速度快但精度略低的模型，以及处理复杂任务能力强但速度稍慢的高端模型。请注意，使用特定高端模型可能需要订阅相应的付费套餐。

### 查看状态
命令 `/status` 用于查看当前登录的账户邮箱、基础配置以及Cloud Code的运行环境状态，帮助用户了解当前的系统工作状态。

### 检查环境
命令 `/doctor` 用于诊断当前系统环境，会列出Node.js版本、Cloud Code安装路径等关键信息。此命令在排查环境配置问题时非常有用，能够快速定位潜在的系统兼容性问题。

## 在集成开发环境中使用

### 插件安装与配置
Cloud Code提供了与主流IDE集成的插件版本。在Cursor或Visual Studio Code的扩展商店中搜索“Cloud Code”，请认准由官方发布的插件，其正确名称为“cloud-code-for-ide”或类似，并具有官方认证标识。安装完成后，IDE界面通常会新增专用的功能按钮或侧边栏入口。

### 集成环境操作方式
点击IDE中的Cloud Code按钮，会在编辑器内部启动一个集成的终端窗口。此窗口的功能与外部独立终端完全一致，支持所有以正斜杠开头的命令操作。集成环境的优势在于能够直接与编辑器中的代码文件联动，例如让AI分析或修改当前打开的文件。部分用户反馈IDE内集成的终端在响应速度上可能略有延迟，您可以根据个人偏好和实际需求选择使用独立终端或集成环境。

## 实战演示：编程问题解决流程

### 创建与测试脚本文件
首先向Cloud Code发出创建文件的指令：“帮我实现一个冒泡排序的Python文件”。AI理解指令后会生成相应的Python脚本文件，并在当前目录中创建该文件。接着可以发出测试指令：“帮我测试上面生成的py文件”。AI会执行该脚本并返回运行结果，在此过程中可能发现各种运行环境问题，例如中文输出乱码等问题。

### 问题诊断与迭代解决
当发现输出存在中文乱码问题时，可以向AI描述具体问题：“上面的bash输出中，中文出现了乱码问题，帮我解决一下”。AI会根据问题描述提供初步解决方案，例如建议将脚本中的中文字符串改为英文字符串。如果此方案不符合需求，可以进一步明确要求：“你的方案确实解决了乱码，但我希望保留中文输出。我需要的解决方案是彻底解决乱码问题，同时保留中文”。

AI会基于更明确的需求进行多轮尝试，可能的方法包括添加系统编码判断与转换逻辑，在Python代码中重定向标准输出流并强制设置为utf-8编码，或者修改环境变量和使用其他编码处理库。每次AI提供修改方案后，都可以要求其运行测试以验证解决方案的有效性。

### 经验总结与最佳实践
这个完整的编程问题解决流程展示了Cloud Code作为编程伙伴的核心价值。它不仅能够生成代码，更能参与调试的全过程，通过多轮迭代找到问题的最佳解决方案。为了获得最高效的合作体验，建议在提问时尽可能明确约束条件和最终目标。当用户已有解决问题的思路时，直接将思路告知AI可以极大缩短迭代过程，实现“人类经验+AI能力”的高效协作模式。这种工作方式特别适合解决复杂的技术问题，能够显著提高开发效率和问题解决质量。
