## 环境准备与安装

Cloud Code 的正常运行依赖于特定的基础软件环境，主要包括 Node.js 运行环境和 Git 版本管理工具。

### 安装 Node.js

请访问 Node.js 官方网站下载安装程序。为确保最佳的兼容性与稳定性，建议安装版本号不低于 20 的 LTS（长期支持）版本。安装完成后，您需要打开终端或命令提示符，并执行 `node -v` 与 `npm -v` 命令。若这两条命令均能返回对应的版本号，则证明 Node.js 环境已正确安装并配置完毕。

### 安装 Git

Cloud Code 的部分功能执行依赖于 Git Bash 终端环境。请下载并安装适用于您操作系统的 Git 版本。安装成功后，在终端中执行 `git --version` 命令，若能显示版本信息，则表明 Git 已准备就绪。

### 安装 Cloud Code 本体

Cloud Code 本身通过 Node.js 的包管理器 npm 进行全局安装。请您根据操作系统，在终端中执行对应的安装命令。
对于 macOS 或 Linux 系统，请使用命令：`sudo npm install -g @cloudcode/cli`。
对于 Windows 系统，请使用命令：`npm install -g @cloudcode/cli`。
安装过程通常较为迅速。若遇到网络延迟，可考虑配置 npm 的国内镜像源以提升下载速度。安装完成后，可通过 `cloud --version` 命令验证安装结果。

## 关键配置步骤

完成基础安装后，需要进行若干关键配置以确保 Cloud Code 能够连接至服务并正常执行任务。

### 配置网络代理

若您的网络环境存在限制，导致直接访问 Cloud 服务时遇到障碍，则需要配置代理服务器。请在您的项目根目录下创建名为 `.cloud` 的文件夹，并在其中创建 `settings.json` 配置文件。该文件内容应遵循以下格式，请将代理地址和端口替换为您实际使用的参数：
```json
{
  "httpProxy": "http://127.0.0.1:7890",
  "httpsProxy": "http://127.0.0.1:7890"
}
```

### 配置 API 密钥

使用 Cloud Code 的核心功能必须提供有效的 API 密钥。您可以选择使用官方密钥或由第三方代理服务提供的密钥。
官方密钥需在 Cloud 官网注册账户并完成相应的服务订阅后获取。第三方服务密钥则需遵循对应服务商的指引获取。
获取密钥后，请将其设置为名为 `CLOUD_API_KEY` 的系统环境变量。若使用第三方服务，可能还需额外设置 `CLOUD_API_BASE` 环境变量以指定其 API 端点地址。

### 配置系统环境变量

为确保 Cloud Code 能够正确调用 Git Bash 终端，请将 Git 安装目录下 `bin` 文件夹的路径（其中包含 `bash.exe`）添加到您系统的 `PATH` 环境变量中。

## 登录授权与验证

所有配置完成后，需通过一次性的登录授权流程来激活工具的使用权限。

### 执行首次登录

在终端中首次执行 `cloud` 命令，将启动交互式配置向导。首先，您需要根据个人偏好选择界面主题（亮色或暗色）。随后，程序将引导您完成身份验证。
若您已预先配置了 `CLOUD_API_KEY` 环境变量，系统会询问是否使用该密钥，请确认使用。若未配置，程序将自动打开默认浏览器，跳转至 Cloud 官方授权页面，请按页面提示完成 OAuth 登录授权。授权成功后，您的访问凭证将自动保存至本地用户目录下的 `.cloud.json` 配置文件中。

### 验证安装成功

完成登录授权后，在终端中再次输入 `cloud` 命令。如果成功进入 Cloud Code 的交互式命令行界面，并显示欢迎信息与命令提示符，即表明 Cloud Code 已完全安装并配置成功，可以开始正式使用。
