# {title}


Cloud Code 提供了多种交互模式，旨在适应不同的使用场景并显著提升工作效率。通过合理切换模式，用户可以实现从自动化代码生成到复杂项目规划的全流程辅助。以下将详细介绍三种核心模式及其应用场景。

## 核心模式概述

Cloud Code 支持通过快捷键（不同版本或系统可能有所差异，请以终端提示为准）在多种工作模式间进行切换。当前版本的主要快捷键为 `Alt+M`。这些模式旨在为用户提供不同级别的自动化与控制权限，以满足从快速执行到周密规划的不同需求。

## 自动编辑模式

自动编辑模式（Auto-Edit Mode）以实现免确认的批量操作为核心特征。在此模式下，当用户提出一个复杂任务（例如“创建一个酷炫的待办事项网站应用”）时，AI 将自动分解任务、生成代码并执行必要的文件操作，而无需在每个步骤等待用户手动确认。整个过程会以颜色标记和动态进度提示的形式在终端中清晰展示，使用户能够直观了解任务当前所处的阶段，例如分析需求、创建 HTML 结构、设计 CSS 样式、实现 JavaScript 功能等。该模式极大地简化了从想法到成品的流程，适合快速原型构建和自动化脚本编写。在使用时，若希望 AI 以中文进行输出和沟通，只需在对话开始时明确说明“请用中文和我沟通”即可。

## 计划模式

计划模式（Plan Mode）是进行项目前期规划和方案梳理的利器。在此模式下，AI 的核心工作是生成详尽的计划文档，而非立即执行。例如，当用户指令为“帮我规划一个贪吃蛇 HTML 小游戏的需求文档、技术栈和实现方案，并输出到 `plan.md` 文件”时，AI 会首先输出一份包含需求分析、技术选型、目录结构设计、核心逻辑流程图等内容的完整方案。用户可以对方案进行评估，若满意则可选择让 AI 自动进入执行阶段，依据该计划文档逐步完成项目开发；若需调整，则可切换至其他模式进行手动编辑。这种“先规划，后执行”的工作流完美模拟了真实的软件开发过程，确保了项目的结构性和可管理性。

## 优乐模式

优乐模式（Yolo Mode）是一种赋予 AI 更高系统权限的运行模式。用户需要在启动 Cloud Code 时通过添加 `--dangerously-skip-permissions` 参数来进入此模式。在该模式下，AI 被授予了执行更多操作的权限，甚至可以执行诸如删除文件等高风险命令，相当于获得了系统的“超级管理员”权限。因此，它通常建议仅在沙箱环境或确保没有重要数据的纯净项目目录中使用。优乐模式适用于大规模代码重构、启动全新复杂项目或修复涉及广泛文件改动的疑难 Bug 等场景，能够最大限度减少人工确认的干预，实现全自动处理。需要注意的是，即使在优乐模式下，用户依然可以通过快捷键切换回其他受约束的模式，以控制权限级别。

## 模式切换与实践建议

不同模式间的切换非常灵活，用户可根据当前任务阶段动态调整。一个高效的工作流可能是：在项目初期使用**计划模式**厘清思路并产出技术方案；随后切换到**自动编辑模式**，依据方案快速生成主体代码和文件结构；若遇到需要大面积、高风险修改的复杂问题时，可在受控环境中短暂启用**优乐模式**来自动化解决。关键在于理解每种模式的设计初衷和风险边界，将 AI 的强大自动化能力与人的决策判断相结合，从而在效率与安全可控之间找到最佳平衡点。
